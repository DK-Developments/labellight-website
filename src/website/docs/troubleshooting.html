Printer Not Detected

If Label Light shows "No printers found" even though your printer is connected, try these solutions:

Solution 1: Check DYMO Connect is Running

Label Light requires DYMO Connect to be running in the background.

  1. Look for the DYMO icon in your system tray (Windows) or menu bar (macOS)
  2. If you don't see it, open DYMO Connect from your applications
  3. In Label Light settings, click the refresh button next to the printer dropdown

Solution 2: Check USB Connection

  1. Ensure your DYMO printer is connected via USB
  2. Try a different USB port
  3. Try a different USB cable
  4. Check if the printer appears in DYMO Connect software

Solution 3: Restart DYMO Web Service

Sometimes the DYMO web service stops responding. To restart it:

On Windows:

  1. Press Win + R to open Run
  2. Type services.msc and press Enter
  3. Find "DYMO Web Service" in the list
  4. Right-click and select "Restart"

On macOS:

  1. Open Terminal
  2. Run: killall DYMO.WebApi.Win
  3. Reopen DYMO Connect

Solution 4: Enable Bypass Printer Detection

Some versions of DYMO Connect have a bug where printer detection always fails. As a workaround:

  1. Open Label Light settings
  2. Enable Bypass Printer Detection in General Settings
  3. Click Save Settings

Only Use When Necessary

Only enable this option if your printer works in DYMO Connect software but isn't detected by Label Light. This bypasses connection checks and may cause errors if the printer is actually disconnected.

DYMO Connect Issues

DYMO Connect Won't Start

If DYMO Connect fails to start or crashes:

  1. Uninstall DYMO Connect completely
  2. Download the latest version from dymo.com/support
  3. Install with administrator privileges
  4. Restart your computer

Web Service Port Conflict

DYMO Connect's web service runs on ports 41951-41954. If another application is using these ports, Label Light won't be able to connect.

To check for port conflicts (Windows):

netstat -ano | findstr "41951"

If another process is using the port, you may need to stop that process or change its port configuration.

HTTPS Certificate Errors

DYMO Connect uses a self-signed certificate for HTTPS. If you see certificate errors:

  1. Ensure you're using the latest version of DYMO Connect
  2. Try reinstalling DYMO Connect
  3. Check your antivirus software isn't blocking the certificate

API Token Errors

"API Token Required" Error

This error appears when trying to print from pages that require API access (like consignments) without a configured token.

Solution: Configure your Lightspeed API token in Label Light settings. See the API Token Configuration guide.

"API Token Expired" Error

Lightspeed API tokens expire periodically (typically after several hours).

Solution:

  1. Get a fresh token from the Chrome DevTools Network tab
  2. Update the token in Label Light settings
  3. Click Save Token

"API Request Failed" Error

This can occur due to network issues or Lightspeed server problems.

Solutions:

  • Check your internet connection
  • Refresh the Lightspeed page and try again
  • Get a fresh API token
  • Wait a few minutes and try again (server may be temporarily unavailable)

Labels Not Printing

Print Job Sent But No Output

If Label Light shows "Print successful" but nothing prints:

  1. Check that labels are loaded in your printer
  2. Check the printer's LED indicators for errors
  3. Open DYMO Connect and try printing a test label
  4. Check if there's a paper jam
  5. Ensure the correct printer is selected in Label Light

Print Queue Stuck

If labels are stuck in the print queue:

On Windows:

  1. Open Settings ‚Üí Devices ‚Üí Printers & Scanners
  2. Click on your DYMO printer ‚Üí Open queue
  3. Cancel all pending jobs
  4. Try printing again

On macOS:

  1. Open System Preferences ‚Üí Printers & Scanners
  2. Select your DYMO printer
  3. Click "Open Print Queue"
  4. Cancel any stuck jobs

Slow Printing

If printing is very slow:

  • Close unnecessary browser tabs
  • Ensure DYMO Connect is the only label software running
  • Try restarting DYMO Connect
  • For large batches, print in smaller groups

Label Formatting Issues

Text Cut Off or Too Small

If text doesn't fit properly on your labels:

  1. Open your label template in DYMO Label Software
  2. Adjust the text object's "Auto Size" settings
  3. Set text to "Shrink to Fit" if content varies in length
  4. Re-upload the template to Label Light

Wrong Label Size

If labels are printing in the wrong position or size:

  1. Ensure your label template matches your physical label size
  2. Check the label type in DYMO Connect matches what's loaded
  3. Try recalibrating the printer (hold the feed button for 5 seconds)

Missing Data on Labels

If some fields are blank on printed labels:

  1. Check your field mappings in Label Light settings
  2. Ensure all label objects have correct mappings
  3. Verify the product has data in those fields in Lightspeed
  4. Check that object names in your template match what Label Light expects

Object Names

Label Light identifies template fields by their object names. If you rename objects in your template, you'll need to update your field mappings.

Extension Not Working

Print Button Not Appearing

If the Label Light print button doesn't appear on Lightspeed pages:

  1. Check that the extension is enabled in Chrome (chrome://extensions)
  2. Refresh the Lightspeed page
  3. Make sure you're logged into Lightspeed
  4. Clear your browser cache and refresh
  5. Disable other extensions temporarily to check for conflicts

Extension Showing Errors

If the extension icon shows an error badge:

  1. Click the extension icon to see the error message
  2. Check that you're signed in
  3. Verify your subscription is active
  4. Try signing out and back in

Reinstalling the Extension

If other solutions don't work, try reinstalling:

  1. Go to chrome://extensions
  2. Find Label Light and click "Remove"
  3. Restart Chrome
  4. Reinstall from the Chrome Web Store
  5. Sign in again

Settings After Reinstall

Reinstalling the extension will reset your settings. You'll need to reconfigure your API token, label templates, and field mappings.

Authentication Issues

Can't Sign In

If you're having trouble signing in with Google:

  1. Ensure you're using the same Google account used for your subscription
  2. Check that pop-ups aren't blocked for the extension
  3. Try signing out of Google in your browser, then signing in fresh
  4. Clear Chrome's cookies and cache

Session Expired

If you keep getting signed out:

  • Your authentication session may have expired - simply sign in again
  • Check that Chrome is set to save cookies
  • Ensure no browser extensions are clearing cookies automatically

Subscription Not Found

If Label Light says your subscription is not active:

  1. Verify you're signed in with the correct email
  2. Check your subscription status at your account page
  3. Contact support if your subscription should be active

Contact Support

If you've tried the solutions above and still need help:

Before Contacting Support

Please have the following information ready:

  • Your Label Light account email
  • Chrome version (find at chrome://version)
  • DYMO Connect version
  • Your operating system (Windows/macOS and version)
  • DYMO printer model
  • Screenshots of any error messages
  • Steps to reproduce the issue

Getting Console Logs

For technical issues, console logs help our team diagnose problems:

  1. Press F12 to open Chrome DevTools
  2. Click the "Console" tab
  3. Look for messages starting with [LL]
  4. Right-click in the console and select "Save as..."
  5. Attach the saved log file to your support request
